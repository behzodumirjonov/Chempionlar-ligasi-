# 🤝 Champions League Hub - Hissa Qo'shish Qo'llanmasi

Projemizga qo'shilganingiz uchun tashakkur! Quyida hissa qo'shish bo'yicha yo'riqnomalar topiladi.

## 📋 Jarayon

1. **Reposni Fork Qiling**
   ```bash
   GitHub saytida "Fork" tugmasini bosing
   ```

2. **Klonlang**
   ```bash
   git clone https://github.com/yourusername/champions-league-hub.git
   cd champions-league-hub
   ```

3. **Branch Yaratang**
   ```bash
   git checkout -b feature/your-feature-name
   ```

4. **O'zgarishlarni Qiling**
   - Kod o'zgarishlari qiling
   - Testlarni o'tkazing
   - Hujjatlarni yangilang

5. **Commit Qiling**
   ```bash
   git add .
   git commit -m "Add: Your feature description"
   ```

6. **Push Qiling**
   ```bash
   git push origin feature/your-feature-name
   ```

7. **Pull Request Yaratang**
   - GitHub saytida "New Pull Request" tugmasini bosing
   - O'zgarishlarni tavsiflab bering
   - Tasdiq uchun yuboring

## 📝 Commit Qo'llanmasi

Commit xabarlari quyidagi shaklda bo'lishi kerak:

```
<type>(<scope>): <subject>

<body>

<footer>
```

### Tiplar

- **feat**: Yangi xususiyat
- **fix**: Xatolarni tuzatish
- **docs**: Hujjat o'zgarishlari
- **style**: Kod formatitlari (no functional change)
- **refactor**: Kodni qayta yozish
- **perf**: Performance yaxshilanishi
- **test**: Test qo'shish yoki o'zgartirish

### Misollar

```
feat(frontend): Add YouTube video integration
fix(backend): Fix CORS issue in API routes
docs: Update README with installation steps
style(components): Format Navigation component code
refactor(pages): Restructure Home page layout
```

## 🎯 Hissa Turları

### 1. Xatolarni Tuzatish (Bug Fixes)
- Issue tanasidan kelib chiqan xatolarni tuzating
- Tuzatilyotgan xatoning tafsilotlarini commit xabarida ko'rsating

### 2. Yangi Xususiyatlar (Features)
- Yangi sahifalar
- Yangi komponentlar
- Yangi API endpoints
- Yangi dizayn elementlari

### 3. Hujjatlar (Documentation)
- README yangilash
- API hujjatlari
- Setup qo'llanmasi
- Deployment hujjatlari

### 4. Performance
- Kodni optimizatsiya qiling
- Bundle hajmini kamaytiring
- API so'rovlarini tezlashtiring

## ✅ Tekshirish Ro'yxati

Pull Request yuborishdan oldin tekshiring:

- [ ] Kod zamonaviy bo'ytirib tuzbalarni qabul qiladi
- [ ] Testlar o'tkazilyapti
- [ ] README yangilandi (agar kerak)
- [ ] Hujjatlari yangilandi
- [ ] Commit xabarlar aniq va tavsiflovchi
- [ ] Xatolar konsolida yo'q
- [ ] Kod izohlandirgan
- [ ] Branchni main ga birlashtirishdagi konflikt yo'q

## 🔍 Code Review

Sizning Pull Request ini review qilish paytida:

1. **Oz'garish tavsifi ko'rib chiqiladi** - O'zgarishlar nima ekanini aniq bo'ladi
2. **Kod sifati** - Code style va best practices
3. **Testlar** - Kodni tekshirish testlar
4. **Dokumentasiya** - Yangilashlar hujjatda ko'rsatilgani
5. **Performance** - Kodni samaraliligi

## 🐛 Bug Report Qilish

Agar bug topsa, issue yaratang:

```markdown
## Bug Tavsifi
Bug nima ekanini tavsiflab bering.

## Takrorlash Bosqichlari
1. Qanday qadamlar bilan bug ko'rinadi?
2. Nima kutilgan edi?
3. Nima bo'ldi?

## Muhit
- OS: Windows/macOS/Linux
- Browser: Chrome/Firefox
- Versin: v1.0.0

## Screenshots
Agar mumkin bo'lsa, screenshot qo'shing
```
